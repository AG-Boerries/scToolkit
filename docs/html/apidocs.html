Modules included:

  • `sc_code` — source Core analysis logic — preprocessing, filtering, clustering, and automated downstream visualizations. Serves as the main entry point for running standardized workflows with robust defaults and integrated error handling.

  • `sc_config` — source Centralized config and parameter handling for all analysis steps. Stores key workflow settings (e.g., dataset name, paths, clustering options) in a standard dictionary structure attached to adata.uns[“config”]. This enables consistent parameter tracking, reproducibility, and versioning without relying on external schema validation libraries.

  • `sc_plots` — source Unified plotting interface for UMAPs, dot plots, violins, heatmaps, ridline, density. Designed for consistency, with shared color handling and auto-labeling across all major marker and enrichment visualizations.

  • `sc_utils` — source Extensive collection of general-purpose utilities for data inspection, stat collection, GPU setup, file system interactions, and result formatting. Includes helpers for safe dictionary access, object copying, marker parsing, random sampling, and unified path management. Central to avoiding duplication and ensuring consistent behavior across the entire codebase.

  • `sc_spatial` — source Support for spatial transcriptomics integration, including polygon parsing, FOV-based partitioning, and spatial plotting extensions.

  • `sc_legacy` — source Deprecated config logic and compatibility layers used to migrate early versions of the framework to the modern, validated config structure.

  • `utils` — source Low-level helper functions — array handling, parsing, normalization, and error-safe lookups.

  • `common_logger` — source Unified logging — structured terminal output, verbosity control, and consistent formatting across CLI and notebook environments.
